# Minimal Fit 개발 로드맵

## 1. 문서 개요

**문서명**: Minimal Fit 개발 로드맵  
**기준 문서**

- `prd.md`
- `designspec.md`

**목적**

PRD 요구사항과 설계서를 기준으로 실제 구현 순서를 정리하고  
GitHub 저장소에서 관리 가능한 개발 로드맵을 정의한다.

---

# 2. 현재 기준 요구사항 범위

현재 PRD 기준 반영 범위

- **REQ-01** Rolling Routine 시작
- **REQ-02** FP 보상
- **REQ-03** 오늘 예정 루틴 확인
- **REQ-04** 운동 통계(PR / 볼륨 / 연속성)
- **REQ-05** 시작 템플릿
- **REQ-06** FP 사용
- **REQ-07** 운동 완료 달력
- **REQ-08** 최소 입력 세트 기록

세트 기록 기능

- 이전 세트 자동 채움
- 원탭 완료
- 세트 복제

설계서 기준 핵심 요소

- `ExerciseInfo` 스키마
- `UserPersistence`
- Zero-Click 세트 동기화
- Rolling Routine 알고리즘
- 템포 가이드

---

# 3. 개발 원칙

1. 코딩 전 `prd.md` 와 `designspec.md` 확인
2. 코드 작성 후 PRD 요구사항 충족 여부 역분석
3. 코드 변경 시 문서 업데이트 제안
4. 커밋 규칙 준수

커밋 형식
- 커밋명 [ver] 제목
- 내용 [commit 주요 내용]

# Phase 1. 프로젝트 골격 및 데이터 기반 확정

## 목표

앱이 최소 동작 가능한 구조를 먼저 만든다.

## 작업 항목

- Next.js 기본 라우팅 구성
- Tailwind / globals.css 설정
- `data/exercises.ts` 구조 정의
- `ExerciseInfo` 타입 정의
- LocalStorage 유틸 작성
- `MINIMAL_FIT_DATA` 초기화

## 완료 기준

- 앱 진입 시 데이터 로드 가능
- 사용자 상태 저장 가능

## 관련 요구사항

- 설계서 2.1 ExerciseInfo
- 설계서 2.2 UserPersistence

## 커밋 예시

커밋명  
`[v0.1.0] 프로젝트 기본 구조 및 저장소 초기화`

내용  
`[commit 주요 내용] Next.js 구조, Tailwind 설정, ExerciseInfo 타입 정의, LocalStorage 상태 초기화`

# Phase 2. Rolling Routine MVP

## 목표

사용자가 고민 없이 바로 운동을 시작할 수 있게 한다.

## 작업 항목

- 홈 화면 `내 루틴 시작하기`
- 마지막 운동 인덱스 로드
- Rolling 알고리즘 적용
- 다음 운동 자동 선택
- 세션 종료 시 인덱스 저장
- 오늘 루틴 미리보기

## 완료 기준

- 마지막 운동 다음 종목 자동 시작
- 오늘 루틴 확인 가능

## 관련 요구사항

- REQ-01
- REQ-03
